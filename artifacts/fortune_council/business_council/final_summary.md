# 命理產品商業討論小組總結

任務ID：CARD-FORTUNE-BUSINESS-COUNCIL-SUMMARY-001
範圍：產品與商業討論，不開發畫面、不開發 API、不做自動化流程

## 1. 先講結論

目前不該問「塔羅、人格、命盤誰是主角」。

比較正確的產品形狀是：

```text
Pantheon 是一個命理陪伴產品。
塔羅、人格、命盤/命書是三種進入方式。
使用者從哪裡進來，取決於他當下想解決什麼問題。
```

所以三個都保留：

| 模組 | 使用者心情 | 產品作用 |
|---|---|---|
| 塔羅 | 我現在有一件事想問 | 當下問題、日常儀式、回訪 |
| 64 人格 | 我想知道我是怎樣的人 | 身份感、分享、社群自我介紹 |
| 命盤/命書 | 我想看人生底色與長期節奏 | 深度信任、付費報告、追問 |

## 2. 這個產品可以長成什麼樣

商業討論小組整理出 4 種合理產品型態。

### 方案 A：命理陪伴社群

一句話：

```text
每天用塔羅互動，用人格建立身份，用命盤/命書承接深問題。
```

適合：

- 想做 Discord / 社群。
- 想要每天有互動。
- 想慢慢養出付費需求。

收入可能來自：

- 付費命書。
- 追問包。
- 會員制。
- 小額每日/每週主題報告。

最大風險：

- 如果日常內容弱，社群會冷。
- 如果只靠抽牌，可能變成普通塔羅工具。

### 方案 B：個人命書產品

一句話：

```text
用人格與塔羅做入口，最後把使用者導向完整個人命書。
```

適合：

- 想比較快看到付費。
- 想做高單價報告。
- 想把內容做成可交付商品。

收入可能來自：

- 個人命書。
- 年度命書。
- 感情 / 事業 / 人際 / 財富 / 人生方向主題命書。
- 付費追問。

最大風險：

- 命盤/命書品質要求最高。
- 八字/紫微目前還是初版能力，不能講太滿。

### 方案 C：自我理解工具

一句話：

```text
先讓使用者用 64 人格理解自己，再用塔羅與命盤補反思。
```

適合：

- 想要低門檻。
- 想讓使用者分享。
- 想吸引不一定相信命理的人。

收入可能來自：

- 進階人格報告。
- 個人特質分析。
- 付費解讀。
- 社群會員。

最大風險：

- 太像一般人格測驗。
- 命理感可能不足。

### 方案 D：問題解決型命理顧問

一句話：

```text
使用者帶著問題來：感情、事業、人際、財富、人生方向，系統用塔羅、人格、命盤分層回應。
```

適合：

- 想做「問問題」的產品。
- 想讓使用者自然追問。
- 想銜接付費諮詢或命書。

收入可能來自：

- 單題付費。
- 追問包。
- 主題報告。
- 深度命書。

最大風險：

- 容易滑向保證答案。
- 感情、事業、財富問題要避免操控、保證結果與恐嚇式銷售。

五個主題：

| 主題 | 典型問題 | 主要承接模組 |
|---|---|---|
| 感情 | 這段關係怎麼看、曖昧/伴侶/婚姻卡在哪 | 塔羅看當下，命盤/命書看長期模式，人格看相處方式 |
| 事業 | 工作選擇、轉職、創業、職場定位 | 命盤/命書看長期節奏，人格看工作模式，塔羅看當下選擇 |
| 人際 | 朋友、同事、社群、界線、被看見 | 人格看互動模式，塔羅看當下關係，命盤/命書看長期課題 |
| 財富 | 賺錢方式、金錢焦慮、穩定感、風險感 | 命盤/命書看資源節奏，人格看決策偏好，塔羅看短期提醒 |
| 人生方向 | 我現在該往哪走、未來重心、人生階段 | 命盤/命書看大方向，塔羅看當下提醒，人格看內在傾向 |

補充：

```text
自我狀態不作第六主題。
它是每個主題底下的解讀角度。
例如感情裡的焦慮、事業裡的卡住、財富裡的不安全感、人生方向裡的迷惘。
```

## 3. 三個模組的正確分工

### 塔羅

負責：

- 當下問題。
- 每日提醒。
- 情緒整理。
- 社群互動。

不負責：

- 證明事情一定會發生。
- 決定使用者人生。
- 回推出生盤因果。

### 64 人格

負責：

- 自我理解。
- 身份感。
- 分享感。
- 社群角色。

不負責：

- 正式心理測驗。
- 診斷人格。
- 覆蓋命盤結論。

### 命盤 / 命書

負責：

- 長期底色。
- 人生節奏。
- 深度報告。
- 付費承接。

不負責：

- 在初版能力下講成完整傳統排盤。
- 保證命運。
- 用恐嚇方式逼使用者付費。

## 4. 可能客群

### 客群 1：想更懂自己的人

可能被什麼吸引：

- 64 人格。
- 自我理解報告。
- 朋友分享。

可能付費原因：

- 想看更完整的自己。
- 想知道感情、事業、人際、財富與人生方向裡的固定模式。

### 客群 2：正在卡一個問題的人

可能被什麼吸引：

- 塔羅。
- 今日提醒。
- 單題反思。

可能付費原因：

- 想追問。
- 想看這個問題背後的命盤因素。

### 客群 3：本來就相信命理的人

可能被什麼吸引：

- 命盤。
- 命書。
- 流年。
- 感情 / 事業 / 人際 / 財富 / 人生方向主題分析。

可能付費原因：

- 想看完整報告。
- 想要個人化解讀。

### 客群 4：社群型使用者

可能被什麼吸引：

- 人格角色。
- 每日抽牌。
- 群內討論。

可能付費原因：

- 會員感。
- 參與感。
- 想被看見、被回應。

## 5. 收費方式

先不要只想一種收費。比較合理的是分層：

| 層級 | 內容 | 收費可能 |
|---|---|---|
| 免費 | 塔羅單題、人格結果、命盤簡介 | 免費換分享 / 進群 / 留問題 |
| 低價 | 單題追問、小主題報告 | 小額付費 |
| 中價 | 感情 / 事業 / 人際 / 財富 / 人生方向主題命書 | 一次性付費 |
| 高價 | 完整個人命書 + 追問 | 高單價或人工審核 |
| 長期 | 社群會員、每月主題、每日儀式 | 訂閱制 |

目前最不該做的是：

```text
一開始就承諾完整自動命書 + 無限追問 + 社群營運 + 每日內容。
```

這會太重。

## 6. 最大風險

| 風險 | 說明 | 控制方式 |
|---|---|---|
| 產品變太散 | 塔羅、人格、命盤都做，使用者不知道你是什麼 | 統一成「命理陪伴產品」 |
| 太像普通測驗 | 64 人格如果沒有命理世界觀，就不特別 | 接上塔羅與命書，但不硬串流程 |
| 太像普通塔羅 | 如果只剩抽牌，差異化不足 | 加入人格與命盤作長期記憶 |
| 命書講太滿 | 初版命盤能力不足時會傷信任 | 強結論必須有依據，不足就寫成反思 |
| 收費太焦慮 | 命理產品容易用焦慮推付費 | 禁止恐嚇式銷售 |
| 社群太早重營運 | Discord 需要持續內容，不是開了就會活 | 先定每日互動格式，不先做大社群 |

## 7. 小組已經能決定的事

```text
1. 不選單一主角。
2. 塔羅、人格、命盤/命書都保留。
3. 三者不是固定流程，而是不同入口與不同用途。
4. 產品大方向應該是「命理陪伴」，不是單純排盤、單純塔羅、單純人格測驗。
5. 收費應分層，不應一開始就做完整高價命書。
6. 下一步不該開發，應先把產品型態收斂。
```
