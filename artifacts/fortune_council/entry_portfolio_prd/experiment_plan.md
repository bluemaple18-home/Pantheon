# Entry Portfolio Experiment Plan

任務ID：CARD-FORTUNE-ENTRY-PORTFOLIO-PRD-001
目的：定義三個可獨立作第一入口的驗證假設，並標出本輪低成本優先驗證，不進工程實作。

## 候選實驗 A：64 人格 Identity / Share

### 假設

年輕華語使用者可以從 64 人格作為第一入口進 Pantheon，先理解自己，並把結果分享給朋友或帶進 Discord。

### 要驗證

- 使用者是否覺得結果「有命中感」。
- 使用者是否願意分享結果。
- 使用者是否願意用結果作為 Discord 角色或自我介紹。
- 使用者是否想看更深的命盤/命書。

### 不驗證

- 不驗證完整命書付費。
- 不驗證塔羅大型牌陣。
- 不驗證正式心理測驗準確度。

### 成功訊號

```text
定性:
- 使用者能用自己的話描述結果為什麼像自己。
- 使用者能指出哪一段最想分享。
- 使用者願意看下一層命理解讀。

定量候選:
- 完成率
- 分享意圖
- Discord 角色意圖
- 想看命盤/命書比例
```

### 失敗訊號

- 使用者覺得它只是普通人格測驗。
- 使用者看完沒有想分享或追問。
- 使用者不理解它和命理產品的關係。

## 候選實驗 B：塔羅 Daily Ritual

### 假設

塔羅可以作為第一入口，尤其適合已經帶著當下問題、情緒或日常儀式需求進來的使用者；它同時也適合回訪與 Discord 每日互動。

### 要驗證

- 使用者是否願意明天再抽一次。
- 使用者是否願意提出當下問題。
- 使用者是否願意把抽牌結果帶到 Discord 討論。

### 不驗證

- 不驗證命運預測。
- 不驗證付費塔羅服務。
- 不驗證完整牌庫與大型牌陣。

### 成功訊號

```text
定性:
- 使用者覺得塔羅提醒能幫他整理當下狀態。
- 使用者願意把問題說清楚。
- 使用者願意回來追蹤。

定量候選:
- 回訪意圖
- 問題提交意圖
- Discord 討論意圖
```

## 候選實驗 C：命盤/命書深度意圖

### 假設

命盤/命書可以作為第一入口，尤其適合已經有高意圖、願意提供出生資料、想看長期底色的使用者；它也可以承接 64 人格或塔羅之後產生的深問題。

### 要驗證

- 使用者是否想看長期底色。
- 使用者是否願意留下深問題。
- 使用者是否願意理解完整命書可能包含什麼。

### 不驗證

- 不驗證完整自動命書交付。
- 不驗證價格。
- 不驗證無上限追問。

### 成功訊號

```text
定性:
- 使用者能說出想用命書解決的具體問題。
- 使用者理解命書不是保證命運，而是結構化反思。

定量候選:
- 深問題提交比例
- 想看完整命書比例
- 願意接受人工/半人工交付比例
```

## 驗證優先序

```text
可作第一入口:
- 64 人格 identity / share
- 塔羅 daily ritual / current question
- 命盤/命書 depth / paid report

本輪低成本優先驗證:
1. 64 人格 identity / share

PM 可改選的優先驗證:
1. 塔羅 daily ritual，如果要先抓當下問題與回訪
2. 命盤/命書深度意圖，如果要先抓高意圖與付費深度
```

原因：

- 64 人格既有能力最成熟，驗證成本最低，所以適合作本輪優先驗證。
- 塔羅不是第二入口；它是問題導向第一入口，只是本輪若看成本可排後。
- 命盤/命書不是第三入口；它是高意圖第一入口，只是完整交付成本較重。
