# Pantheon 第一階段產品形狀決策稿

任務ID：CARD-FORTUNE-PRODUCT-SHAPE-DECISION-001
狀態：recommend
範圍：產品與商業形狀，不開發畫面、不開發 API、不做自動化流程

## 1. 建議結論

Pantheon 第一階段建議定義為：

```text
一個圍繞五大人生主題的命理陪伴產品。
```

五大主題固定為：

```text
感情 / 事業 / 人際 / 財富 / 人生方向
```

塔羅、64 人格、命盤/命書不是誰當主角，而是分工：

| 模組 | 負責什麼 |
|---|---|
| 塔羅 | 回答使用者當下問題，形成每日互動與回訪 |
| 64 人格 | 建立自我理解、身份感、分享與社群角色 |
| 命盤/命書 | 承接長期底色、人生節奏、深度報告與付費 |

## 2. 主敘事

建議主敘事：

```text
Pantheon 幫你把感情、事業、人際、財富、人生方向裡卡住的問題，整理成可理解、可反思、可追問的命理陪伴。
```

不要主打：

```text
最準算命
命運保證
AI 大師
完整自動命書
```

原因：

- 「最準」會把產品推進高風險承諾。
- 「AI 大師」會讓信任變薄。
- 「完整自動命書」目前交付成本太重。

## 3. 產品骨架

第一階段產品應該有三層，不是一條固定流程。

### 第一層：免費入口

用途：讓使用者開始說出問題或看見自己。

可包含：

- 塔羅單題。
- 64 人格結果。
- 命盤簡介。
- 五大主題選擇。

重點：

```text
免費層只要讓使用者覺得「這跟我有關」，不需要一次講完整人生。
```

### 第二層：追問與主題報告

用途：承接使用者已經在意的主題。

可包含：

- 感情小報告。
- 事業小報告。
- 人際互動報告。
- 財富與安全感報告。
- 人生方向報告。
- 單題追問。

重點：

```text
這一層適合低價或中價，不承諾完整命書。
```

### 第三層：完整命書 / 深度服務

用途：給已經有深問題、願意付費的人。

可包含：

- 完整個人命書。
- 年度命書。
- 主題命書。
- 有上限的追問包。

重點：

```text
這一層可以高價，但必須控制交付範圍與品質。
```

## 4. 五大主題怎麼用

| 主題 | 免費層可以問 | 付費層可以做 |
|---|---|---|
| 感情 | 這段關係現在卡在哪？ | 感情模式、伴侶相處、關係週期報告 |
| 事業 | 我現在的工作方向怎麼看？ | 職場定位、轉職節奏、創業適性報告 |
| 人際 | 我在人際裡常卡在哪？ | 社交模式、界線、被看見與歸屬感報告 |
| 財富 | 我對金錢的焦慮來自哪裡？ | 資源節奏、風險偏好、賺錢方式反思報告 |
| 人生方向 | 我接下來的重心是什麼？ | 年度方向、人生階段、長期節奏命書 |

自我狀態不獨立成主題，放進每個主題底下：

```text
感情裡的焦慮
事業裡的卡住
人際裡的不安
財富裡的安全感
人生方向裡的迷惘
```

## 5. 建議收費形狀

先採分層，不要一開始只做高價命書。

| 層級 | 商品 | 價格感 |
|---|---|---|
| 免費 | 單題塔羅 / 64 人格 / 命盤簡介 | 免費 |
| 低價 | 單題追問 / 小主題解讀 | 小額 |
| 中價 | 五大主題報告 | 中價 |
| 高價 | 完整命書 / 年度命書 / 追問包 | 高價 |
| 長期 | 會員 / 每月主題 / 社群活動 | 訂閱或會員 |

第一階段最合理的付費商品不是完整命書，而是：

```text
五大主題小報告。
```

原因：

- 比完整命書輕。
- 比單題塔羅更有價值感。
- 能測哪個主題最有付費意圖。
- 也能回頭餵完整命書設計。

## 6. 短期不做

```text
不做完整自動命書。
不做無上限追問。
不做大型塔羅牌陣。
不做醫療、法律、投資建議。
不做恐嚇式銷售。
不把 64 人格寫成心理診斷。
不把命盤結果寫成保證命運。
不急著做常駐 agent team。
```

## 7. 這個方向的優點

- 不需要選塔羅、人格、命盤誰是主角。
- 五大主題讓產品不會散。
- 免費入口、低價報告、高價命書可以自然分層。
- 塔羅能做回訪，人格能做分享，命盤/命書能做付費深度。
- 之後要做社群、報告、追問，都有清楚位置。

## 8. 最大風險

| 風險 | 說明 | 降低方式 |
|---|---|---|
| 五大主題太像一般算命分類 | 使用者可能覺得普通 | 每個主題都要用 Pantheon 的語氣：整理、反思、追問，不是保證答案 |
| 免費層太散 | 塔羅、人格、命盤都出現會混亂 | 免費層只要讓使用者選主題與入口，不一次講完 |
| 付費層太重 | 完整命書太早會壓垮內容品質 | 先做五大主題小報告 |
| 社群太早啟動 | 沒有每日內容會冷 | 先定每日塔羅 / 主題問題格式，不急著正式營運 |
