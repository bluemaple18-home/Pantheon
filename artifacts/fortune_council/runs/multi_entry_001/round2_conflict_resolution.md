# Round 2 Conflict Resolution

用途：只處理 Round 1 的分歧，不展開完整討論。

## 主要分歧

```text
分歧 1: 三個入口是否一定要分第一、第二、第三？
分歧 2: 如果三個都可作第一入口，本輪低成本優先驗證該選誰？
分歧 3: 命盤是否應作高意圖第一入口，還是先留在深度層？
```

## 收斂判斷

### 分歧 1：三個入口是否一定要排序

判斷：不應排序成固定第一、第二、第三。64 人格、塔羅、命盤/命書都可以獨立作第一入口。

理由：
- 64 人格抓「我是誰」。
- 塔羅抓「我現在有一件事想問」。
- 命盤/命書抓「我想看人生底色」。
- 這三種是不同使用者起心動念，不是同一條漏斗的前後步驟。

### 分歧 2：本輪低成本優先驗證該選誰

判斷：若只看成本與既有完成度，優先驗證 64 人格 identity/share；若 PM 想抓當下問題與回訪，也可以優先驗證塔羅。

理由：
- 64 人格已有 64 分支與 UnifiedReport 回流，成本最低。
- 塔羅的吸引力高、儀式感強、適合每天回來。
- 優先驗證順序是資源決策，不是產品入口階級。

### 分歧 3：命盤是否可作第一入口

判斷：可以。命盤/命書適合高意圖使用者作第一入口，但本輪不宜把完整命書交付做重。

理由：
- 命盤讓 Pantheon 不只是人格測驗或抽牌工具。
- 但八字/紫微目前仍是 MVP scaffold，強結論需要證據鏈約束。
- 本輪可先驗證深度意圖，不必承諾完整自動命書。

## 可接受的降級版本

```text
第一階段不做單一大流程。
先把三個入口定義成入口 portfolio：

1. 64 人格: identity / share / community role
2. 塔羅: daily ritual / current question / Discord interaction
3. 命盤/命書: depth / trust / paid report

入口關係:
64 人格、塔羅、命盤/命書都可作第一入口。

本輪低成本優先驗證建議:
先驗證 64 人格作為低摩擦身份入口，原因是既有完成度最高。

替代優先驗證:
若 PM 想先抓當下問題與回訪，改先驗證塔羅 daily ritual。
若 PM 想先抓高意圖與付費深度，改先驗證命盤/命書深度意圖。
```

## PM 需要決定

- 本輪優先驗證要選 64 人格、塔羅，還是命盤/命書。
- 是否接受「三個都可作第一入口」的入口 portfolio。
- 若選命盤/命書優先，是否接受只驗證深度意圖，不承諾完整自動交付。
