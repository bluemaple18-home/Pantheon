# Pantheon 公開文章展出規範

## 1. 定位

Pantheon 的公開文章不是傳統運勢新聞，也不是完整個人命書。

本規範接在既有內容矩陣之後使用：

- SEO / AEO / GEO 矩陣：`artifacts/fortune_council/content_seo_matrix/`
- Click108 近兩年研究證據：`output/click108_research/report.md`
- 前台文章 policy 來源：`app/web/static/article-registry.js`
- 前台 SEO JSON-LD 實作：`app/web/static/article-seo.js`

公開文章的任務是：

- 回答搜尋問題。
- 幫讀者把情境整理清楚。
- 說明塔羅、人格、命盤、星盤各自能看什麼。
- 保留限制，不給個人化結論。
- 把讀者收攏到文章入口或五大主題文章。

核心邊界：

```text
文章給答案，但不給「你的答案」。
公開文章給理解。
文章入口整理問題。
個人判讀要回到讀者自己的資料與情境。
```

## 2. 可借鑑 Click108 的部分

Click108 近兩年文章的優點是「好掃、好懂、好點」。Pantheon 可以借鑑形式，不照抄承諾。

可借鑑：

- 標題有明確對象、情境、結果。
- 第一段快速進入讀者關心的問題。
- 文章用固定模板，讓讀者知道往下會看到什麼。
- 小標密集，手機上容易掃讀。
- 語氣溫和，不把低運寫成恐嚇。
- 每段都落到生活情境：感情、工作、人際、金錢、安全感、方向。
- 結尾有下一步，而不是只停在知識。

不可借鑑：

- 不寫「誰最旺」「一定轉運」「財路大開」這類結果承諾。
- 不用大量推廣連結打斷正文。
- 不把命理寫成預言。
- 不用恐懼、焦慮或保證結果換點擊。

## 3. Pantheon 寫字風格

Pantheon 的文章要像一個冷靜、懂命理、但不替讀者下判決的顧問。

語氣標準：

- 白話，短句，先講結論。
- 具體，不寫玄而空的漂亮話。
- 可以溫柔，但不能哄騙。
- 可以指出風險，但不能恐嚇。
- 可以給整理方式，但不替讀者做人生決定。
- 每篇至少有一段「這不能代表什麼」。

避免詞：

```text
全面解析
深度解析
快速變化的時代
不可或缺
賦能
不僅
更是
總而言之
值得注意的是
必看
一定
保證
注定
```

推薦句型：

```text
比較準確的看法是...
這不代表...，比較像是在提醒...
如果你現在卡在...，可以先分成三層看。
公開文章只能講通用意思；個人狀況還要放回你的問題和資料裡。
```

## 4. SEO / AEO / GEO 必備欄位

每篇文章上稿前都要有這些欄位：

| 欄位 | 規範 |
|---|---|
| 主攻關鍵字 | H1 和第一段前 80 字一定出現 |
| 次要關鍵字 | 至少 2 個 H2 或段落自然出現 |
| 長尾問句 | 放入 FAQ 3 到 5 題 |
| meta title | 28 到 36 個中文字優先，含主關鍵字 |
| meta description | 70 到 95 字，說明適用情境與限制 |
| answer 段 | 50 字內短答案，給 AEO 使用 |
| 導言 | 150 字內完整回答，第一屏就能理解 |
| tags | 必含產品線、情境、公開文章邊界 |
| canonical | 指向正式文章 URL |
| JSON-LD | Article、FAQPage、BreadcrumbList |
| CTA | 導向文章入口或五大主題文章 |

固定標籤：

```text
Pantheon
繁體中文
公開文章
通用知識
SEO
AEO
GEO
```

## 5. 標題規範

標題要同時服務搜尋與點擊，不要只做情緒鉤子。

推薦公式：

```text
[主關鍵字] 是什麼？[讀者在意的面向] 怎麼看
[工具/主題] 怎麼看 [情境]？先整理 [數字] 個問題
[牌/宮位/人格] 意思：[正位/逆位/感情/事業] 怎麼看
[情境] 卡住怎麼辦？[工具1]、[工具2] 與 [工具3] 可以看什麼
```

可用範例：

```text
感情塔羅怎麼問？復合、曖昧、關係卡住怎麼看
事業運勢怎麼看？轉職、創業、工作卡住的整理方式
紫微斗數是什麼？命盤十二宮和主星怎麼看
月亮星座是什麼？情緒與安全感怎麼看
```

不使用：

```text
這 3 種命格下半年一定翻身
財運爆發！你是不是被點名的人？
抽到這張牌代表他一定會回來
命盤有這顆星注定感情失敗
```

## 6. 每篇固定文章結構

```text
H1: 含主攻關鍵字的主標題
導言: 150 字內直接回答問題
快速答案: 50 字內，給搜尋摘要與答案引擎
H2: 這個問題通常在問什麼
H2: 這個工具可以看什麼
H2: 放到五大情境裡怎麼理解
H2: 常見誤解與不能代表什麼
H2: 如果想看自己的狀況，下一步怎麼做
FAQ: 3 到 5 題短問短答
延伸閱讀: 同分類、跨分類、主題入口
CTA: 文章入口或主題文章
```

文章長度：

- 基礎概念文：1,200 到 1,800 字。
- 單牌、單宮位、單人格文：1,300 到 2,000 字。
- 五大情境問題文：1,600 到 2,400 字。
- 總整理頁：2,400 到 3,500 字。

段落規則：

- 每段 80 到 160 字。
- 手機上避免連續大段文字。
- 每個 H2 下至少 2 段，最多 4 段。
- 表格只用在比較、清單、工具分工，不用來堆裝飾。

## 7. 產品線對應

| 產品線 | 文章目的 | 可以借 Click108 的形式 | Pantheon 必須補上的邊界 |
|---|---|---|---|
| 塔羅 | 解釋牌義、提問方式、當下問題整理 | 牌組分段、情境提醒、短段落 | 塔羅不是預言，單牌不能決定結果 |
| 人格 | 解釋 MBTI、16 型、64 分支與互動模式 | 類型分段、感情/工作/人際拆解 | 人格不是診斷，不是永久標籤 |
| 命盤 | 解釋八字、紫微、宮位、主星、流年 | 宮位/主星分組、生活情境化 | 單一宮位或星曜不能做完整判斷 |
| 星座 | 解釋星盤、太陽/月亮/上升與星座落點 | TOP/月份/關係主題的可讀性 | 不寫固定命運，不寫星座決定一切 |

## 8. 五大情境模板

### 感情

文章要處理：曖昧、復合、關係卡住、安全感、相處模式。

固定角度：

- 先分清楚是溝通、期待、安全感，還是未來方向。
- 塔羅看當下狀態。
- 人格看相處偏好。
- 命盤看長期關係課題。
- 不承諾復合，不替對方下結論。

CTA：

```text
如果你想知道這段關係放到你的狀況裡比較像哪一種，可以先用感情文章入口整理問題，再讀更接近情境的主題文章。
```

### 事業

文章要處理：轉職、工作卡住、創業、被看見、職場人際。

固定角度：

- 先分清楚問題是能力、環境、資源、節奏，還是方向。
- 塔羅看短期卡點。
- 人格看工作方式。
- 命盤看長期事業節奏。
- 不保證升職、轉職成功或創業賺錢。

### 人際

文章要處理：界線、社交疲憊、溝通、被看見、關係變淡。

固定角度：

- 先分清楚是互動模式、界線、角色期待，還是情緒負擔。
- 人格看反應模式。
- 塔羅看當下互動盲點。
- 命盤看長期人際課題。
- 不鼓勵控制別人。

### 財富

文章要處理：金錢焦慮、資源節奏、花錢模式、收入與安全感。

固定角度：

- 先分清楚是收入、支出、風險感、資源配置，還是安全感。
- 命盤看資源節奏。
- 人格看冒險與消費偏好。
- 塔羅看短期提醒。
- 不提供投資建議，不承諾賺錢。

### 人生方向

文章要處理：迷惘、選擇、轉換期、該動或該等。

固定角度：

- 先分清楚是害怕、疲憊、不適合，還是資訊不足。
- 塔羅看當下選項。
- 人格看選擇偏好。
- 命盤看階段主題。
- 不替讀者做重大人生決策。

## 9. 展出版型規範

### 文章列表 / 首頁

桌機：

- 首屏保留一個主推文章區。
- 右側或旁邊放熱門/最新/主題入口。
- 下方用產品線或五大情境分區。
- 每張文章卡必有：分類、標題、摘要、標籤。

手機：

- 單欄排列。
- 主推文章第一屏要能看完標題與摘要。
- 熱門/最新改成排名或短列表。
- 卡片摘要不超過 2 到 3 行。

### 文章頁

桌機：

- 主文寬度約 680 到 720px。
- 右側欄放延伸閱讀、產品線入口與五大情境入口。
- 標題下顯示分類、更新日期、閱讀時間、公開文章邊界。
- 第一屏要有導言與快速答案，不要讓廣告或裝飾推開正文。

手機：

- 單欄閱讀，主文寬度約 350px。
- 標題可以多行，但不要壓縮行距。
- 快速答案放在導言後，讓讀者先得到答案。
- CTA 不要每幾段重複，正文中段最多一次，結尾一次。

## 10. 內部連結規範

每篇至少要有：

```text
1. 產品線入口：命盤 / 人格 / 塔羅 / 星座
2. 五大情境入口：感情 / 事業 / 人際 / 財富 / 人生方向
3. 同分類相關文章 2 篇
4. 跨分類相關文章 1 到 2 篇
5. 文章入口或主題文章 CTA
```

錨文字用關鍵字，不用「點這裡」。

範例：

```text
延伸閱讀：塔羅牌正位逆位是什麼、感情塔羅怎麼問、人際關係卡住怎麼辦
```

## 11. FAQ 規範

FAQ 要回答真問題，不寫漂亮但沒有資訊的句子。

每篇 3 到 5 題：

- 1 題定義。
- 1 題限制。
- 1 題情境應用。
- 1 題和產品入口有關。
- 需要時補 1 題風險或誤解。

範例：

```text
Q: 塔羅牌可以看復合嗎？
A: 可以整理這段關係目前卡在哪裡，但不能保證對方一定會回來。比較好的問法是：我現在能看見什麼盲點，下一步怎麼做比較清楚？
```

## 12. 收攏 CTA 規範

CTA 要承接文章，不要硬賣。

固定句型：

```text
如果你只是想理解這個概念，這篇文章已經足夠。
如果你想知道它放到你自己的狀況裡代表什麼，可以先選一個文章入口：

1. 看塔羅文章入口，整理當下問題
2. 看人格文章入口，整理反應模式
3. 看命盤文章入口，了解長期底色

如果你已經有明確問題，再看感情 / 事業 / 人際 / 財富 / 人生方向主題文章。
```

不同產品線可替換第一句：

- 塔羅：單張牌只能提供一個反思角度。
- 人格：類型只能說明常見偏好，不是你的完整人格。
- 命盤：單一宮位只能說明一個觀察角度。
- 星座：單一星座落點只能說明一部分。

## 13. 上稿檢查表

發布前逐項確認：

- H1 有主攻關鍵字。
- 第一段前 80 字有主攻關鍵字。
- 前 150 字直接回答搜尋問題。
- 有 50 字內快速答案。
- 至少 2 個 H2 自然放入次要關鍵字。
- 有 3 到 5 題 FAQ。
- 有「不能代表什麼」的限制段。
- 有產品線入口、情境入口、相關文章內鏈。
- 有文章入口或主題文章 CTA。
- meta title、description、canonical、Article JSON-LD、FAQ JSON-LD 都完整。
- 沒有保證、恐嚇、醫療、法律、投資建議。

## 14. 一篇合格文章的骨架

```text
H1: 感情塔羅怎麼問？復合、曖昧、關係卡住怎麼看

導言:
感情塔羅適合用來整理一段關係現在卡在哪裡，但不適合用來保證復合或控制對方。比較好的問法，是把問題拆成當下狀態、自己的盲點、下一步選擇。

快速答案:
感情塔羅可以看關係狀態與行動提醒，但不能保證對方想法或最終結果。

H2: 感情塔羅通常在問什麼
H2: 復合、曖昧和關係卡住要分開看
H2: 塔羅可以看什麼，不能看什麼
H2: 人格和命盤可以補上哪些脈絡
H2: 如果想看自己的狀況，下一步怎麼做
FAQ
延伸閱讀
CTA
```
