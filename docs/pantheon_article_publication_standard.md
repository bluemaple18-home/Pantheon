# Pantheon 公開文章展出規範

## 1. 定位

Pantheon 的公開文章不是傳統運勢新聞，也不是完整個人命書。

本規範接在既有內容矩陣之後使用：

- SEO / AEO / GEO 矩陣：`artifacts/fortune_council/content_seo_matrix/`
- Click108 近兩年研究證據：`output/click108_research/report.md`
- Click108 寫法重研究 v2：`artifacts/fortune_council/content_seo_execution/evidence/click108_voice_research_v2.md`
- 前台文章 policy 來源：`app/web/static/article-registry.js`
- 前台 SEO JSON-LD 實作：`app/web/static/article-seo.js`

公開文章要做到：

- 回答搜尋問題。
- 先處理讀者正在卡住的煩惱，再補工具語言。
- 幫讀者把感情、工作、人際、財富或人生方向的情境整理清楚。
- 說明塔羅、人格、命盤、星盤能輔助看哪一層，不把工具本身寫成課程。
- 保留限制，不給個人化結論。
- 讓讀者讀完知道自己現在要確認哪個條件、哪個限制或哪個下一步。

核心邊界：

```text
文章給答案，但不給「你的答案」。
公開文章給理解。
問題先於工具。
個人判讀要回到讀者自己的資料與情境。
```

## 2. 可借鑑 Click108 的部分

Click108 近兩年文章的優點是「好掃、好懂、好點」。Pantheon 可以借鑑形式，不照抄承諾。

可借鑑：

- 標題有明確對象、情境、結果。
- 第一段快速進入讀者關心的問題。
- 文章用固定模板，讓讀者知道往下會看到什麼。
- 小標密集，手機上容易掃讀。
- 語氣溫和，不把低運寫成恐嚇。
- 每段都落到生活情境：感情、工作、人際、金錢、安全感、方向。
- 結尾自然收束，讓讀者知道文章回答到哪裡，不靠獨立「下一步」卡片補救。
- 不把讀者當成塔羅老師、命理師或 MBTI 研究者；他們是來處理煩惱，不是來上工具課。

不可借鑑：

- 不寫「誰最旺」「一定轉運」「財路大開」這類結果承諾。
- 不用大量推廣連結打斷正文。
- 不把命理寫成預言。
- 不用恐懼、焦慮或保證結果換點擊。

### 2.1 去模板化規則（v2）

Pantheon 不能只把 Click108 的「好掃」翻成更多固定小標。先前 44 篇塔羅小牌文章共用多組完整句型，讀者會感覺只是替換牌名。新版要求每篇都先寫出該牌或主題的具體生活場景，再進入工具語言。

- 開頭第二句要出現該篇專屬的困擾或情境，不得連續使用「查 X 的人通常不是想背……」作為開場。
- 每篇至少使用 3 個具體動詞，例如等訊息、守住現金流、補作品集、談分工、收尾、試探、拒絕或停下來。
- 正位、逆位、感情、工作與限制段不得只套同一套結尾；限制句要接在具體解讀後面。
- 同一個固定完整句型在同一批文章最多出現 3 次；任兩篇的前兩段不得只替換關鍵字。
- 每篇至少有 2 個專屬場景、2 個可觀察動詞、1 個「這張工具也可能不適用」的反例。
- 保留主關鍵字、FAQ、answer、JSON-LD、canonical 與內鏈規範；去模板化只改文字，不改版面或 SEO 契約。

例：不要先寫「查錢幣八意思的人通常不是想背牌義」，先寫「會查錢幣八意思，常常是因為你已經做了一陣子，卻還看不出成果到底有沒有累積」。讀者先被自己的問題接住，再進入牌義，文章才不會像工具說明書。

## 3. Pantheon 寫字風格

Pantheon 的文章要像一個冷靜、懂命理、但不替讀者下判決的顧問。

語氣標準：

- 白話，短句，先講結論。
- 具體，不寫玄而空的漂亮話。
- 可以溫柔，但不能哄騙。
- 可以指出風險，但不能恐嚇。
- 可以給整理方式，但不替讀者做人生決定。
- 每篇至少有一段「這不能代表什麼」。

避免詞：

```text
全面解析
深度解析
快速變化的時代
不可或缺
賦能
不僅
更是
總而言之
值得注意的是
必看
一定
保證
注定
```

推薦句型：

```text
比較準確的看法是...
這不代表...，比較像是在提醒...
如果你現在卡在...，可以先分成三層看。
公開文章只能講通用意思；個人狀況還要放回你的問題和資料裡。
```

禁止正文主軸：

```text
先教讀者學會塔羅、命盤、人格或星盤，再要求讀者自己套用。
把「78 張牌、正逆位、牌陣順序」寫成文章收束主軸。
把「文章入口、標籤頁、集結頁、五大主題文章」這類站方管理語言寫給讀者看。
```

## 4. SEO / AEO / GEO 必備欄位

每篇文章上稿前都要有這些欄位：

| 欄位 | 規範 |
|---|---|
| 主攻關鍵字 | H1 和第一段前 80 字一定出現 |
| 次要關鍵字 | 至少 2 個 H2 或段落自然出現 |
| 長尾問句 | 放入 FAQ 3 到 5 題 |
| meta title | 28 到 36 個中文字優先，含主關鍵字 |
| meta description | 70 到 95 字，說明適用情境與限制 |
| answer 段 | 50 字內短答案，給 AEO 使用 |
| 導言 | 150 字內完整回答，第一屏就能理解 |
| tags | 必含產品線、情境、公開文章邊界 |
| canonical | 指向正式文章 URL |
| JSON-LD | Article、FAQPage、BreadcrumbList |
| 導讀 | 用正文收束、上一篇/下一篇、延伸閱讀與 topic 內鏈完成 |

固定標籤：

```text
Pantheon
繁體中文
公開文章
通用知識
SEO
AEO
GEO
```

## 5. 標題規範

標題要同時服務搜尋與點擊，不要只做情緒鉤子。

推薦公式：

```text
[主關鍵字] 是什麼？[讀者在意的面向] 怎麼看
[工具/主題] 怎麼看 [情境]？先整理 [數字] 個問題
[牌/宮位/人格] 意思：[正位/逆位/感情/事業] 怎麼看
[情境] 卡住怎麼辦？[工具1]、[工具2] 與 [工具3] 可以看什麼
```

可用範例：

```text
感情塔羅怎麼問？復合、曖昧、關係卡住怎麼看
事業運勢怎麼看？轉職、創業、工作卡住的整理方式
紫微斗數是什麼？命盤十二宮和主星怎麼看
月亮星座是什麼？情緒與安全感怎麼看
```

不使用：

```text
這 3 種命格下半年一定翻身
財運爆發！你是不是被點名的人？
抽到這張牌代表他一定會回來
命盤有這顆星注定感情失敗
```

## 6. 每篇固定文章結構

```text
H1: 含主攻關鍵字的主標題
導言: 150 字內直接回答問題
快速答案: 50 字內，給搜尋摘要與答案引擎
H2: 這個問題通常在問什麼
H2: 這個工具可以看什麼
H2: 放到五大情境裡怎麼理解
H2: 常見誤解與不能代表什麼
H2: 如果想看自己的狀況，先把問題縮小
FAQ: 3 到 5 題短問短答
延伸閱讀: 只放推薦文章，不放產品線入口或情境入口
不放獨立 CTA 區塊
```

文章長度：

- 基礎概念文：1,200 到 1,800 字。
- 單牌、單宮位、單人格文：1,300 到 2,000 字。
- 五大情境問題文：1,600 到 2,400 字。
- 總整理頁：2,400 到 3,500 字。

段落規則：

- 每段 80 到 160 字。
- 手機上避免連續大段文字。
- 每個 H2 下至少 2 段，最多 4 段。
- 表格只用在比較、清單、工具分工，不用來堆裝飾。

## 7. 產品線對應

| 產品線 | 文章目的 | 可以借 Click108 的形式 | Pantheon 必須補上的邊界 |
|---|---|---|---|
| 塔羅 | 整理感情、工作、選擇卡住時的當下問題 | 情境提醒、短段落、避免好壞二分 | 塔羅不是預言，單牌不能決定結果，也不是要教讀者學會塔羅 |
| 人格 | 解釋 MBTI、16 型、64 分支與互動模式 | 類型分段、感情/工作/人際拆解 | 人格不是診斷，不是永久標籤 |
| 命盤 | 解釋八字、紫微、宮位、主星、流年 | 宮位/主星分組、生活情境化 | 單一宮位或星曜不能做完整判斷 |
| 星座 | 解釋星盤、太陽/月亮/上升與星座落點 | TOP/月份/關係主題的可讀性 | 不寫固定命運，不寫星座決定一切 |

## 8. 五大情境模板

### 感情

文章要處理：曖昧、復合、關係卡住、安全感、相處模式。

固定角度：

- 先分清楚是溝通、期待、安全感，還是未來方向。
- 塔羅看當下狀態。
- 人格看相處偏好。
- 命盤看長期關係課題。
- 不承諾復合，不替對方下結論。

正文收束句：

```text
如果你想知道這段關係放到你的狀況裡比較像哪一種，先把問題寫成一句具體描述：你在等對方表態、害怕失去，還是不知道自己要不要繼續。
```

### 事業

文章要處理：轉職、工作卡住、創業、被看見、職場人際。

固定角度：

- 先分清楚問題是能力、環境、資源、節奏，還是方向。
- 塔羅看短期卡點。
- 人格看工作方式。
- 命盤看長期事業節奏。
- 不保證升職、轉職成功或創業賺錢。

### 人際

文章要處理：界線、社交疲憊、溝通、被看見、關係變淡。

固定角度：

- 先分清楚是互動模式、界線、角色期待，還是情緒負擔。
- 人格看反應模式。
- 塔羅看當下互動盲點。
- 命盤看長期人際課題。
- 不鼓勵控制別人。

### 財富

文章要處理：金錢焦慮、資源節奏、花錢模式、收入與安全感。

固定角度：

- 先分清楚是收入、支出、風險感、資源配置，還是安全感。
- 命盤看資源節奏。
- 人格看冒險與消費偏好。
- 塔羅看短期提醒。
- 不提供投資建議，不承諾賺錢。

### 人生方向

文章要處理：迷惘、選擇、轉換期、該動或該等。

固定角度：

- 先分清楚是害怕、疲憊、不適合，還是資訊不足。
- 塔羅看當下選項。
- 人格看選擇偏好。
- 命盤看階段主題。
- 不替讀者做重大人生決策。

## 9. 展出版型規範

### 文章列表 / 首頁

桌機：

- 首屏保留一個主推文章區。
- 右側或旁邊放熱門/最新/主題入口。
- 下方用產品線或五大情境分區。
- 每張文章卡必有：分類、標題、摘要、標籤。

手機：

- 單欄排列。
- 主推文章第一屏要能看完標題與摘要。
- 熱門/最新改成排名或短列表。
- 卡片摘要不超過 2 到 3 行。

### 文章頁

桌機：

- 主文寬度約 680 到 720px。
- 桌機主文維持舒適閱讀寬度；可見延伸閱讀放在正文與 FAQ 後方的底部模組，不擠壓主文欄寬。
- 延伸閱讀用純文字列表，不做重卡片；可包含上一篇 / 下一篇、分類文章入口與推薦文章，但不得放五大情境入口或工具入口。
- 上一篇 / 下一篇併入底部延伸閱讀清單，不加大灰色外框，不占滿整個版面焦點。
- 標題下顯示分類、更新日期、閱讀時間、公開文章邊界。
- 第一屏要有導言與快速答案，不要讓廣告或裝飾推開正文。

手機：

- 單欄閱讀，主文寬度約 350px。
- 標題可以多行，但不要壓縮行距。
- 快速答案放在導言後，讓讀者先得到答案。
- 單篇文章不放獨立「下一步」CTA 區塊；正文最後一節要自然收束。

## 10. 內部連結規範

每篇至少要有：

```text
1. 可見延伸閱讀：最多 6 條，包含上一篇 / 下一篇、分類文章入口與推薦文章
2. 推薦文章：最多 5 篇真實文章，最多 2 篇同分類，其餘跨分類不重複
3. 內文 topic 連結：最多 8 個，只連本篇主攻或次攻 SEO 標籤
4. 入口導覽：產品線入口、情境入口或工具入口放在頁首、頁尾、產品線頁或獨立導覽
```

錨文字用關鍵字，不用「點這裡」。

範例：

```text
延伸閱讀：塔羅牌正位逆位是什麼、感情塔羅怎麼問、人際關係卡住怎麼辦
```

## 11. 文章知識庫與流水號規範

目前文章知識庫以靜態前端 registry 管理，不另外開 CMS。正式 source of truth 是：

```text
app/web/static/article-registry.js
app/web/static/article-meta.js
app/web/sitemap.xml
tests/test_web.py
```

`article-registry.js` 負責管理：

- `ARTICLE_SERIAL_REGISTRY`：內部文章 ID 對公開流水號。
- `TOPIC_REGISTRY`：常用標籤的 topic ID、英文 slug、中文 label、aliases。
- `ARTICLE_URL_CONTRACT`：公開 URL 契約。
- `getArticlePath()`：產生正式文章 URL。
- `listArticlesForTopic()`：從 topic 反查相關文章。

`article-meta.js` 負責管理：

- `ARTICLE_BODY_LIBRARY`：正式文章內文。
- `buildArticleContent()`：文章頁、產品線頁、topic 集結頁的內容輸出。
- `displayTagLinks`：公開可見標籤與 `/topics/...` 超連結。

公開文章 URL 固定使用英文分類流水號：

```text
/articles/personality/personality-0001
/articles/tarot/tarot-0001
/articles/fortune/fortune-0001
/articles/astrology/astrology-0001
/articles/love/love-0001
/articles/career/career-0001
/articles/interpersonal/interpersonal-0001
/articles/wealth/wealth-0001
/articles/life-direction/life-direction-0001
```

流水號原則：

- 分類名稱全英文，不使用中文。
- 號碼固定 4 位數，從 `0001` 開始。
- 同一分類往後累加，不重用舊號。
- 舊語意 slug 只能做相容轉址，不再作為 canonical。
- canonical、sitemap、文章卡片、延伸閱讀與上一篇/下一篇都必須使用流水號 URL。

topic 集結頁 URL 固定使用英文 slug：

```text
/topics/mbti
/topics/16-personalities
/topics/fortune
/topics/bazi
/topics/ziwei-doushu
/topics/astrology
/topics/love
/topics/career
/topics/interpersonal
/topics/wealth
/topics/life-direction
```

topic 管理原則：

- topic 內部 ID 使用 `topic-0001`、`topic-0002`。
- 公開 URL 使用英文 slug，不使用流水號，保留搜尋可讀性。
- `label` 是讀者看到的中文或英文詞。
- `aliases` 放常見搜尋詞與同義詞，用來讓文章標籤自動連回 topic。
- 常用公開標籤必須有 topic 頁，不能只顯示成死文字。
- 內部標籤如 `SEO`、`AEO`、`GEO`、`公開文章`、`通用知識` 不應顯示在公開標籤列。

文章內標籤規則：

- 公開標籤顯示給自然流量讀者看，必須像正常文章導覽，不像內部營運 metadata。
- 有 topic 對應的標籤要輸出成 `/topics/...` 連結。
- 正文段落裡打到本篇公開 SEO 標籤詞時，也要在不影響閱讀的前提下轉成 `/topics/...` 內文連結。
- 正文內文連結每篇最多 8 個；不屬於本篇主攻或次攻 SEO 標籤的詞，不自動連結。
- 沒有 topic 的詞可以暫時顯示純文字，但不能成為主要導覽。
- 每篇文章至少要有一個可點擊 topic 標籤。
- topic 頁使用 `CollectionPage` JSON-LD；單篇文章使用 `Article` JSON-LD。

延伸閱讀推薦規則：

- 可見延伸閱讀最多 6 條，可包含上一篇 / 下一篇、分類文章入口與推薦文章。
- 推薦文章只能放真實文章連結，不能放「產品線入口」、「情境入口」、「主題入口」或工具入口。
- 單篇文章的推薦文章最多 5 篇。
- 同分類文章最多 2 篇，優先選主題、搜尋詞、產品線與 section 最相關的文章。
- 剩下最多 3 篇跨分類文章，且每篇必須來自不同分類。
- 推薦文章顯示文字只用正常文章標題，不顯示流水號。
- 流水號只保留在 URL、canonical、sitemap、上一篇/下一篇順序導覽和內部管理資料。
- 上一篇/下一篇是流水號順序導覽，可併入可見延伸閱讀，但不算進推薦文章數量。
- 產品線入口可用分類文章入口放在可見延伸閱讀；情境入口、工具入口若需要保留，應放在頁首、頁尾、產品線頁或獨立導覽，不得混在推薦文章清單。

當文章量擴到數百篇以上，應把 registry 拆成資料檔，但公開契約不變：

```text
data/articles/*.json
data/article_bodies/*.md
data/topics.json
```

拆檔只是維護方式改變，不得改變正式 URL、topic URL、canonical、sitemap 或舊 slug 轉址。

## 12. FAQ 規範

FAQ 要回答真問題，不寫漂亮但沒有資訊的句子。

每篇 3 到 5 題：

- 1 題定義。
- 1 題限制。
- 1 題情境應用。
- 1 題和產品入口有關。
- 需要時補 1 題風險或誤解。

範例：

```text
Q: 塔羅牌可以看復合嗎？
A: 可以整理這段關係目前卡在哪裡，但不能保證對方一定會回來。比較好的問法是：我現在能看見什麼盲點，下一步怎麼做比較清楚？
```

## 13. 收攏與導讀規範

單篇文章不放獨立「下一步」CTA 區塊。文章如果需要再用一整塊 CTA 解釋讀者下一步，代表正文、FAQ、上一篇/下一篇或延伸閱讀沒有完成導讀責任。

單篇文章的收攏方式：

```text
1. 正文最後一節把限制、適用情境與下一個可讀問題說清楚。
2. 上一篇 / 下一篇放在 FAQ 上方，負責同分類順序導讀。
3. 延伸閱讀只放推薦文章，負責跨文章導讀。
4. 產品線入口、情境入口或工具入口只出現在頁首、頁尾、產品線頁或獨立導覽，不放在單篇文章底部。
```

產品線頁可以保留「從哪篇開始」導覽，因為它本來就是列表入口；單篇文章不得使用「下一步」卡片補救閱讀路徑。

- 塔羅：單張牌只能提供一個反思角度。
- 人格：類型只能說明常見偏好，不是你的完整人格。
- 命盤：單一宮位只能說明一個觀察角度。
- 星座：單一星座落點只能說明一部分。
